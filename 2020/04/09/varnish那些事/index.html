缓存

HTTP常见的缓存头

  • Expires：http1.0推出，服务器返回文件有效期。
  • Last-Modified：http1.0推出，浏览器会带上 if-modified-since 向服务器发送请求，并和Last-Modified做对比，时间不同则返回200 相同则返回304。
  • Cache-Control：http1.1推出，文件缓存的有效时间。max-age单位是 s，no-cache不会被缓存，no-store不允许被存储，public缓存可以被多用户共享，private单用户私有。优先级高于 Expires。
  • Etag：http1.1推出，该版本号是服务端生成的，浏览器会带上 if-none-match向服务器发送请求，与服务器上的Etag做对比，版本号不同返回200 相同则返回304。
  • Age：消息头里包含消息对象在缓存代理中存贮的时长，以秒为单位。

​ 优先级： 使用上 http1.1 > http1.0

​ 优先级肯定是优先本地，所以 Expires > Last-Modified , Cache-Control > Etag

Varnish相关(官网)

一、问题处理集锦

  1. 报错信息：session_linger has been renamed to timeout_linger* and it is in seconds now (previously was milliseconds).

    解决方法：ldconfig 之后才能执行 varnishncsa

  2. 报错信息：出现大量503错误

    解决方法：

    ​ (1)、查看服务器是否达到最大连接数

    ​ a. 调整服务器的 max open file 最大值

    ​ b. 查看 tcp 参数是否优化过(比如:TIME WAIT 是否做了数量限制等)

    ​ (2)、varnish 连接数是否达到了最大

    ​ a. 查看当前 varnish 最大可处理请求数，增加参数大小 thread_pool_max

    ​ b. 增加进程 max open file 数值

  3. 报错信息：Can’t open log - retrying for 5 seconds

    ​ Can’t open VSM file (Cannot open /varnish/gaea97/_.vsm: No such file or directory）

    解决方法：mkdir /varnish/gaea97

    ​ ln -s /dev/shm/.vsm /varnish/gaea97/.vsm
